# 布布时光机 · 奶油马卡龙视觉升级融合计划

> 来源：Claude Design 导出的「奶油马卡龙」视觉原型（`布布时光机.zip`，style1 一套主题）。
> **铁律：只升级视觉 / 风格 / UI / 动效 / 主题，绝不删减或调整任何现有结构、功能、数据、页面。**
> 换皮不换骨。星座点亮 = 在现有里程碑功能之上加视觉表现，不动里程碑的数据与逻辑。
> 本文档只分析与规划，不含任何代码改动。

---

## 0. 一句话结论

现有 `BubuTheme` 和马卡龙设计**本就同源**（都是奶油粉珊瑚暖色系），所以这不是推倒重来，
而是「**把现有色板调浓一档 + 注入新的视觉质感层（柔光球/闪烁星/玻璃Tab/星座盘/渐变Hero）**」。
全部通过**新增视觉组件 + 微调 token** 实现，业务逻辑、页面结构、数据模型、功能一律不碰。

## 1. 设计语言对照（设计稿 → 现有）

| 维度 | 马卡龙设计稿 | 现有 iOS | 结论 |
|---|---|---|---|
| 主色 rose | `#F2789F` | primary `#F28C9E` | ✅ 几乎一致，可直接对齐 |
| 强调 deeprose | `#E15C86` | 无 | 🆕 新增「按钮渐变深色端」 |
| 奶油底 cream | `#FFF7F1` | cream `#FCF6EC` | ✅ 微调更亮一点点 |
| 文字 ink | `#5A3D34` | warmBrown `#5C473D` | ✅ 一致 |
| 次文字 ink2 | `#A98D82` | secondaryText | ✅ 一致 |
| 辅助色 peach/lav/mint/butter/sky | 一套马卡龙副色 | 仅 success/info/warning | 🆕 补一组「记忆色相 hue」副色板 |
| 圆角 | 28–30（更圆） | card 22 / button 28 | 🔧 卡片圆角加大到 ~28 |
| 字体 | 圆体粗字（800/900） | rounded（已圆体） | ✅ 同方向，标题字重再加粗 |

**核心判断**：色板同源，改动是「**加浓 + 补副色 + 加圆**」，不是换色系。深色模式现有 token 已完备，新增视觉层要顺带做深色适配。

## 2. 设计稿带来的「新增视觉元素」（现有没有，要新建为可复用组件）

这些是马卡龙风格的灵魂，全部做成**独立视图组件**，哪里想要哪里用，不改任何业务视图的逻辑：

| 新组件 | 作用 | 对应设计稿 |
|---|---|---|
| `BubuBlobBackground` | 柔光大色块氛围背景（模糊球） | `MBlob` |
| `BubuSparkle` / `BubuSparkleField` | 闪烁小星点缀（动画） | `MSparkle` |
| `BubuHeroCard` | 渐变 Hero 卡（peach→pink→lav）+ 头像 + 大数字 | 首页 Hero |
| `BubuGlassTabBar` | 玻璃态胶囊底栏 + 中央凸起记录键 | `MTabBar` |
| `BubuProgressRing` | 进度环（已有类似，可统一升级） | `MRing` |
| `BubuConstellationView` | **成长星座盘**（里程碑可视化，见 §4） | `MacMilestones` |
| `BubuDreamPhoto` | 照片占位的梦幻奶油渐变（无图时更美） | `MPhoto` |

> 这些组件**不替换**现有任何视图，而是作为「可选皮肤层」被现有视图**调用/包裹**。

## 3. 全局视觉升级（逐处，纯视觉）

> 原则：每一处都是「现有视图保持结构和数据不变，只换外观修饰符 / 包一层视觉组件」。

| 现有视图 | 视觉升级（不动功能） |
|---|---|
| **全局背景** `ThemedBackground` | 在现有渐变之上叠 `BubuBlobBackground`（2–3 个柔光球）+ 极淡 `BubuSparkleField`，营造马卡龙氛围。深色模式降透明度。 |
| **首页** `CaptureHomeView` | 顶部「☀︎ 早安呀 + 名字 + 年龄」问候行（纯展示）；身份卡区域升级为 `BubuHeroCard` 渐变质感（数据照旧）；统计双卡加圆角/柔影。 |
| **身份卡** `BubuIdentityCard` | 背景渐变换成 peach→pink→lav，加 1–2 颗 `BubuSparkle`，翻面动画保留。 |
| **底部 Tab** `RootTabView` | 换成 `BubuGlassTabBar` 玻璃胶囊样式 + 中央凸起「＋记录」键（Tab 项、跳转逻辑完全不变）。 |
| **里程碑** `MilestonesHomeView` | 渲染层升级为「成长星座盘」（见 §4），数据/点亮逻辑不动。 |
| **时光轴 / 详情 / 相册** | 卡片圆角加大、柔影统一、无图时用 `BubuDreamPhoto`；列表入场动画保留。 |
| **健康 / 胶囊 / 故事** | 同步卡片质感（圆角/柔影/副色），结构不变。 |
| **按钮** | 主按钮统一 rose→deeprose 渐变 + 按压缩放（现有 clickEffect 保留）。 |
| **token** `BubuTheme` | 注入：deeprose、一组 hue 副色、卡片圆角 22→28。**只加不改删**，老引用全部兼容。 |

## 4. 里程碑「成长星座」融合（重点）

### 你的两个诉求
1. **不改里程碑的结构和功能** —— 数据模型 `Milestone`、点亮逻辑、达成判定全不动。
2. **里程碑数量要足够丰富** + **有星座点亮效果**。

### 设计稿的星座方案（已读懂）
- 里程碑摆成**星座盘散点布局**；已达成 = 发光大星（径向渐变 + 光晕 `macHalo` 动画 + 投影），未达成 = 小灰点（虚线圈）。
- **已点亮的星之间连线**成星座（`linePath`）。
- 顶部进度环「已点亮 N / 总数 颗星」。
- 每颗星带 `hue` 色相 + `icon` 符号；点亮的星可点进详情（复用现有里程碑详情）。

### 融合做法（纯视觉，零功能改动）
| 项 | 做法 |
|---|---|
| **渲染层** | 新增 `BubuConstellationView`：读现有 `[Milestone]`，`isAchieved` → 发光星，未达成 → 灰点；已达成的星按顺序连线。**点击星 = 调用现有里程碑点亮/详情逻辑**，行为完全不变。 |
| **保留旧视图** | 现有的「奖章网格」可作为「列表视图」保留，星座盘作为「星空视图」，加一个视图切换（或直接星座盘替换展示层）。**数据同一份**。 |
| **里程碑数量丰富** | 这是**数据层的内容扩充**，不是结构改动 —— 在现有里程碑「预置清单」里**增加更多预置里程碑条目**（大运动/精细动作/语言/认知/社交/生活自理/情感 等分类，几十条）。模型字段不变，只是预置数据更多。每条带 `hue` 色相和符号，让星座盘更繁星满天。 |
| **点亮动效** | 点亮瞬间：星从灰点 → 放大发光 + 光晕扩散 + 连线延伸到新星 + 轻 haptic + 吉祥物 cheer（现有 symbolEffect/ceremony 可复用）。 |
| **星座 + 星座** | 设计稿首页有 `MacMiniConstellation` 迷你星座预览卡 —— 首页可加一张「成长星座 已点亮 N/M」入口卡。 |

> ⚠️ 澄清：设计稿里 `sign: 双鱼座` 是**生日星座**（占星）。你说的「星座点亮」其实是**成长里程碑星座盘**（每个第一次点亮一颗星）。两者不冲突：身份卡可顺带展示生日星座（双鱼座）作为静态信息，里程碑页是可点亮的成长星座盘。

## 5. 桌面小组件同步升级（你特别要求考虑）

小组件已有小/中/大三尺寸 + 头像 + 暖色渐变。马卡龙化升级（纯视觉）：

| 尺寸 | 升级 |
|---|---|
| Small | 背景换 peach→pink 渐变 + 1 颗 sparkle；头像圆形描边保留。 |
| Medium | 加迷你星座预览（已点亮 N 颗）或渐变 Hero 风。 |
| **Large** | 做成「马卡龙身份卡」：渐变背景 + 头像 + 名字/年龄 + **迷你成长星座**（已点亮的小星点阵）。 |

> 约束同主 App：小组件**不跑持续动画**（sparkle 在小组件里用静态版），按天刷新不变。
> 小组件配色目前是硬编码 `WidgetPalette`，升级 = 改那几个常量 + 加渐变，结构不变。

## 6. 动效升级（在现有动效之上，不替换）

| 动效 | 来源 | 落点 |
|---|---|---|
| 闪烁小星 twinkle | `macTwinkle` | 背景/Hero/身份卡点缀（低频、reduceMotion 关闭） |
| 星光晕 halo | `macHalo` | 星座盘已点亮的星 |
| 数字卷动 countUp | `useCountUp` | 「相遇第N天」「已点亮N颗」（现有 numericText 已有，统一） |
| 进度环填充 | MRing transition | 星座进度环 |
| 按压缩放 | Press scale | 已有 clickEffect，统一参数 |
| 卡片入场 | — | 现有 transition 保留 |

> 性能纪律不变：**交互/数据驱动为主，背景动效极低频，绝不重蹈持续满负荷重绘**。

## 7. 实施分阶段（建议，纯视觉）

| 阶段 | 内容 | 风险 |
|---|---|---|
| **V1 主题层** | BubuTheme 注入 deeprose/hue 副色/圆角；新增 BubuBlob/Sparkle/DreamPhoto/HeroCard/GlassTabBar/ProgressRing 组件 | 低（只增不改） |
| **V2 全局换皮** | 首页/身份卡/Tab/背景/按钮/卡片套用新组件与质感 | 低-中（逐屏，结构不动） |
| **V3 成长星座** | BubuConstellationView + 里程碑渲染层升级 + 点亮动效；扩充预置里程碑数据（更多条目） | 中（新视图，数据只增） |
| **V4 小组件马卡龙化** | 三尺寸渐变 + sparkle + 迷你星座 | 低（独立 extension） |
| **V5 深色模式 + 适老复核** | 新视觉层的深色适配；最大字号不破版 | 低 |

> 建议先做 **V1+V2**（主题层 + 全局换皮，立竿见影、零功能风险），**V3 成长星座**单独一轮（含里程碑数据扩充），**V4 小组件**收尾。

## 8. 明确不做的事（守住铁律）

- ❌ 不删任何页面、Tab、功能、字段。
- ❌ 不改数据模型结构（里程碑「数量丰富」= 预置数据增多，不是改 schema）。
- ❌ 不改同步、账号、存储逻辑。
- ❌ 不改导航结构（Tab 还是那几个，只是 Tab 栏外观变）。
- ✅ 只动：颜色、圆角、阴影、渐变、背景氛围、字重、动效、里程碑「展示方式」、预置里程碑「内容条数」。

## 9. 需要你确认的点（动手前）

1. **里程碑展示**：星座盘**替换**现有奖章网格，还是**新增一个「星空视图」**与网格并存切换？（建议并存，给用户选）
2. **里程碑扩充到多少条**：要我按「大运动/精细/语言/认知/社交/自理/情感」分类补到约 40–60 条预置里程碑吗？（数据更多，星空更满）
3. **范围**：先做 iOS（V1+V2）看效果，还是 iOS + 鸿蒙同步换皮？（鸿蒙你在改，建议 iOS 先行）
4. **深浅**：全量换成马卡龙，还是作为「可切换的新主题」与现有主题并存？（现有有多主题系统，可作为新增一款主题）
