# 布布时光机 · 下一波升级计划（创新 · 互动 · 零门槛 · 全家设备）

> 日期：2026-07-07 ｜ 基线：main@b4dc7f3（小组件高级化 + 绘本重构 + 服务端 11 迁移已全部上线）
> 主题四个词：**大胆创新、增加互动、降低操作难度、多设备联动**
> 原则不变：离线优先、自托管、隐私至上、"姥姥能用"是验收标准
> 每条都标了 🔨工作量（S=1天内 / M=2-4天 / L=1周+）和依赖，可直接丢给 Opus 分批执行

## 0. 一句话总纲

现在的布布时光机是一个"**要打开才想起来用**"的 App。下一波的目标是把它变成"**长在全家生活里**"的东西：
抬腕说一句就记下（Watch）、照片拍了自动想收（相册挂载）、姥姥看到大按钮就会点（极简模式）、
客厅电视自动放布布这一年（相框模式）、想不起来的事问一句就有答案（布布问答）。

## 1. 零操作记录 —— 把"记录"从任务变成顺手

> 现状：记录要 解锁手机 → 找 App → 点加号 → 选类型。四步，姥姥经常走丢。
> 目标：**最常见的记录路径压到 0~1 步。**

### 1.1 🥇 Apple Watch App「抬腕一句话」 🔨L
全家戴表的人抬腕 → 按一下 → 说话 → 放下手，一条带语音+转写的记录就进了时光机。
- **watchOS 独立 target**（SwiftUI），只做三件事：① 大按钮录音记录 ② 今日概览（年龄/今日已记几条）③ 喂奶/睡觉一键打卡
- 数据通路：WatchConnectivity 传回 iPhone → 走现有 EntryWriter/同步（不直连服务器，省电省复杂度）
- **表盘复杂功能（Complications）**：陪伴天数 / 生日倒计时 / 一键记录入口，用现有 `SharedWidgetSnapshot` 数据
- 转写：先扔给现有 `/transcribe`（自托管 Whisper），iOS 26 的端侧 SpeechAnalyzer 作为二期替换
- 验收：从抬腕到记录落库 ≤10 秒；断网时存本地、回连自动同步

### 1.2 🥇 相册自动挂载「今天拍了布布」 🔨M
90% 的记录素材本来就在系统相册里。App 不该让人"再选一遍"。
- 每次打开 App（或每晚提醒时），用 Photos framework 扫当天新照片 → Vision 人脸聚类匹配布布 → 首页弹一张温柔卡片：「今天拍了 12 张布布，挑 3 张收进时光机？」一键全收/挑选
- 人脸匹配完全端侧（Vision FaceprintGeneration），不出设备，符合隐私铁律
- 顺手解决"照片在相册、故事在 App"的割裂——这是同类 App（Qeepsake 等）的核心留存功能
- 验收：授权后首页出现建议卡；一键收录带 EXIF 时间/地点自动填

### 1.3 🥈 分享扩展（Share Extension） 🔨M
微信里家人发来布布视频、相册里翻到老照片 → 系统分享面板直接「存进布布时光机」，不用切 App。
- Share Extension target + 现有 App Group 存储 + EntryWriter，架构现成
- 验收：从相册/微信分享一张图，打开 App 已在时光轴

### 1.4 🥈 通知直接回复 =「今日一问」零打开 🔨S
每晚的"今日一问"推送，**在通知上直接打字回答**（UNTextInputNotificationAction）→ 后台落一条记录。
全家最低成本的每日仪式：锁屏上答一句，一年攒 365 条。
- 现有 ReminderScheduler + EntryWriter 都在，只差 notification action 接线

### 1.5 🥉 Action Button / 控制中心一键录音 🔨S
现有 `BubuRecordControl` 只是打开 App。升级成**按下即开始录音**的 `AudioRecordingIntent`（iOS 18 支持后台录音 intent），松手保存。侧边键一按就录，全程不亮屏。

## 2. 互动升级 —— 让全家人"玩"起来

> 现状：记录是单向的（写→存）。反应只有"亲一下"，家人合奏藏得深。
> 目标：**每条记录都能引发一次家庭互动。**

### 2.1 🥇 布布问答（RAG 问答助手） 🔨L ⭐本波最大创新
把三年的记录变成**能对话的记忆**：
- "布布第一次叫妈妈是什么时候？" "她 6 个月的时候多重？" "去年今天我们在干嘛？"
- 实现：服务端加一个 `/ask` 接口——把问题 + 检索到的相关记录（SQLite FTS5 全文检索 entries/milestones/healthrecords，按时间和关键词取 top 20 条）拼进 DeepSeek prompt，答案带"依据的记录"卡片可点跳
- 入口：魔法屋顶部一个对话框 + Siri（"问问布布时光机……"，App Intent 已有底座）
- 隐私：检索在自家服务器，只发文字给 DeepSeek（与现有 AI 一致）；可切本地 Ollama
- 验收：三类问题（时间点/数值/模糊回忆）各答对，且能给出处

### 2.2 🥇 语音反应 + 表情雨 🔨M
- 反应从单一"亲一下"扩成 6 种（❤️😂😭👏🎂⭐），**长按反应可以录 3 秒语音**（"哈哈哈这个表情绝了"）——姥姥不爱打字但爱说话
- 收到新反应时，记录详情页来一场对应 emoji 的**表情雨动效**（复用 BubuBurst 粒子）
- 数据：Comment 的 Reaction 机制已有哨兵设计，扩枚举 + voiceFile 已支持

### 2.3 🥈 家庭接龙故事 🔨M
每周日晚推送一张老照片给全家：「这张照片的故事，每人补一句」→ 各自的一句话按顺序编成一段合奏文，AI 润色成一篇小故事，自动收进绘本。
- 复用：FamilyEnsemble（多视角评论）+ 现有 `/rewrite` AI + 绘本(inStorybook)
- 这是"家人合奏"从功能变成**仪式**的关键一步

### 2.4 🥈 成长电影升级：真成片 + 家人配音 🔨L
- 服务端加 ffmpeg 合成：照片序列 + Ken Burns 参数 + 配乐 + AI 旁白 TTS → 真 MP4，可存相册、发家族群（现在只是 App 内幻灯片，分享不出去=白做）
- 大胆的点：**旁白可以选"妈妈的声音读"**——录 1 分钟样本用开源 TTS（GPT-SoVITS，自托管）克隆，给未来的布布听妈妈 25 岁的声音讲她 1 岁的事
- 验收：一键生成 60 秒年度电影 MP4 并分享成功

### 2.5 🥉 绘本"读给布布听" 🔨S
绘本阅读器加 TTS 朗读模式（AVSpeechSynthesizer 端侧，零成本）：晚上抱着娃翻绘本，App 轻声读，翻页跟随。给"用户是未来的布布"这个定位第一个**现在就能用**的场景。

## 3. 降低操作难度 —— 姥姥模式

> 现状：功能越加越多，首页 5 个入口、魔法屋 6 个功能。对老人是迷宫。

### 3.1 🥇 极简模式（大字大按钮） 🔨M
设置里一个开关「极简模式」：整个 App 变成**三个全屏大按钮**——📷 拍一张 / 🎤 说一段 / 📖 看布布，字号 1.5 倍，无 tab、无二级菜单。切身份为姥姥/爷爷时默认开启。
- 实现：RootView 分叉一个 `SimpleModeView`，全部复用现有 EntryWriter/Timeline 组件
- 这是"姥姥能用=验收标准"的终极兑现，也是同类产品没有的差异化

### 3.2 🥇 Dynamic Type 收尾 + 全量大字号回归 🔨M
上轮改造过 BubuTheme，但要按上上轮 review 清单做**最大辅助字号档的逐页回归**（四宫格→单列、TabBar 标签、时光轴卡片），确保系统字体调到最大不破版。

### 3.3 🥈 首页"下一步建议"卡 🔨S
首页顶部一张智能卡，永远只给**一个**最该做的事："今天还没记录，拍一张？"／"体重 30 天没量了"／"胶囊后天解锁！"。老人不需要选择，跟着卡片走就行。规则引擎本地跑，不用 AI。

## 4. 多设备联动 —— 布布无处不在

### 4.1 🥇 Apple Watch（见 1.1，本波多设备的头牌）

### 4.2 🥇 iPad/旧手机 =「布布相框」 🔨M
家里的 iPad 或退役 iPhone 插上电，变成厨房/客厅的**电子相框**：
- App 加"相框模式"（横屏全屏轮播：精选照片 + 今日年龄 + 那年今日 + 新记录实时飘入）
- 数据走现有同步，天然实时；防烧屏轮播 + 夜间自动调暗
- 一行代码的价值：`UIApplication.shared.isIdleTimerDisabled = true` + 现有 GrowthMoviePlayer 的 Ken Burns 组件复用
- 这让"自托管服务器"第一次有了**全家可见的实体存在感**

### 4.3 🥈 Apple TV「客厅放映厅」 🔨L
tvOS target：沙发上全家一起看年度成长电影、翻星座里程碑、放照片墙。周末晚上的家庭仪式。
- 优先级低于相框（相框零新硬件、tvOS 要新 target + 遥控交互适配），但情感价值最高，适合作为 v2.0 的"发布会功能"

### 4.4 🥈 CarPlay 音频：「路上听布布」 🔨M
开车时听"成长之声"（按岁归档的语音）和 AI 日记朗读。CarPlay audio app 模板 + 现有 VoiceMemo 数据。通勤路上想娃的真实场景。

### 4.5 🥉 HomePod/Siri 全屋记录 🔨S
"嘿 Siri，用布布时光机记一笔：布布今天会拍手了"——`RecordMomentIntent` 已存在，需要真机调通 Siri 短语识别率 + 文档教全家怎么说。零新代码，纯打磨接线。

### 4.6 联动底座：家庭推送 🔨M（4.x 的共同依赖）
"爸爸刚发了 3 张照片"要能**推**到妈妈手机/手表上，现在只有 30 秒轮询。
- 方案：自托管 ntfy（mini 上跑，Docker 一行）+ PocketBase hook 在新记录时发通知 + App 端 ntfy WebSocket/轮询桥接本地通知
- 不碰 APNs 证书体系（自托管纯内网即可达，Tailscale 内全设备可达）；Watch 通知随 iPhone 转发免费获得

## 6. 明确不做的（守边界）

- ❌ 公开社交/分享到小红书类功能——这是家庭私域，泄露面只减不增
- ❌ 安卓/Web 端——鸿蒙版已覆盖非苹果设备，精力聚焦
- ❌ 订阅/付费——自家用的东西
- ❌ 第三方分析 SDK / 云服务依赖——自托管铁律，ntfy/Ollama/GPT-SoVITS 全部跑在 mini 上
