# 布布时光机 · 二期升级改造计划

> 写于 2026-06-11。基于当前代码（Wave A–H 完成、DESIGN_UPGRADE P0 已落地、P1 进行中）与 `screenshots_xhs/` 截图的真实状态。
> 本文回答一个问题：**怎样把一个「功能齐了但还有初期产品味」的 App，升级成温暖、流畅、高级、家人天天想打开的传家产品。**
>
> 与既有文档的关系：
> - `HANDOFF.md` —— 架构地图与验证方法，改代码前必读，本文不重复；
> - `DESIGN_UPGRADE.md` —— 视觉/动效专项，其 P1 余项与 P2 整体**吸收进本文 Wave I/J 排期**，今后以本文为唯一路线图；
> - 北极星不变：**「高级育儿日记本」的质感 + 姥姥能用的清晰 + 给 18 岁布布看的庄重感**。

## 0. 现状诊断 —— 「初期产品味」到底出在哪

逐页走查截图 + 代码后，把「简陋感」拆解成五个可动手的来源（后文五个方向一一对应）：

| # | 来源 | 具体表现 | 对应章节 |
|---|------|---------|---------|
| 1 | **主题是「换色」不是「换世界」** | `ThemeManager.swift` 里每套主题只有 `primary/secondary/背景渐变` 三个值；切到薄荷绿，卡片、填充、发丝线仍是暖棕系；背景是平直双色渐变，无质感层 | §2 主题与质感 2.0 |
| 2 | **图片管线裸奔，滚动掉帧** | `MediaThumbnail.swift`：无内存缓存（每次 cell 复用都重读磁盘+重解码）、`UIImage(data:)` 全尺寸解码、缺缩略图时直接解原图、远端用无缓存的 `AsyncImage`；`TimelineView.groupedSections` 是计算属性，每次 body 求值都全量重分组 | §3 流畅度工程 |
| 3 | **设置页是「管理后台」** | `SettingsView.swift`：系统 Form 平铺 9 个 Section，服务器地址/密码/API Key 这些一年碰一次的字段和「布布的档案」平起平坐；`LabeledContent` 式同步状态像运维面板 | §5.1 设置页重构 |
| 4 | **互动是单向的** | 家人之间只有「合奏补充」和评论；没有轻量的「亲一下」、没有每日话题、没有周报、没有 Widget——打开 App 之外没有任何触点 | §4 互动系统 |
| 5 | **功能有骨架缺血肉** | 健康记录无成长曲线、记录多了没有搜索、「那年今日」只有通知没有页面、备份健康度没人盯、胶囊还不是真 E2E | §5 功能完善 |

**总判断：地基（架构/离线优先/设计 token/动效 token）是好的，不推翻任何东西。这一期的主题词是：渗透（主题渗到每个像素）、丝滑（帧率当指标管理）、合理（信息架构按使用频率重排）、连接（让记录这件事有来有回）。**

## 1. 升级总纲

五个方向，对应五个 Wave（延续 A–H 命名）：

```
方向                Wave   预估     一句话目标
─────────────────────────────────────────────────────────────
流畅度工程          I      1.5 周   任何一页滚动、任何一个动效，120Hz 不掉帧
主题与质感 2.0      J      1.5 周   每套主题是一个完整世界，第一眼「哇」
互动系统            K      2 周     家人之间有来有回，App 外也有触点
功能完善与合理化    L      2.5 周   设置像家、健康有曲线、记录可搜索、数据有兜底
远期储备            M      不排期   真成片/Realtime/端侧转写/多娃
```

> 顺序有讲究：**先把帧率和图片管线修好（Wave I）再做新视觉（Wave J）**——否则更华丽的主题会放大卡顿；互动（K）依赖 Wave J 的贴纸/声音资产；功能（L）独立性强，可与 K 并行穿插。

每个 Wave 的硬验收沿用 `HANDOFF.md` 的标准（clean build 零警告、单测全过、reduceMotion 全流程走查、深色+6 主题截图轮），本文在各节追加专项验收。

## 2. Wave J · 主题与质感 2.0 —— 「更漂亮、更高级」

### 2.1 主题模型升级：从 3 个值到完整语义包

`BubuThemeDefinition` 扩展为完整主题包（**这是本方向一切工作的前提**）：

```swift
struct BubuThemeDefinition {
    // 现有
    let id, name: String
    let primaryHex, secondaryHex: String
    let backgroundStyle: BackgroundStyle

    // 新增 —— 让主题渗进每个像素
    let surfaceTintHex: String        // 卡片/softFill/hairline 的色相偏移基色（DESIGN_UPGRADE 2.1 遗留项）
    let meshPalette: [String]         // 首页头部 MeshGradient 的 4–6 个控制点颜色
    let accentGradient: (String, String)  // 主按钮/进度环的渐变对
    let paperTexture: PaperTexture    // .none / .grain(细颗粒) / .fiber(纤维纸纹)
    let tabTintHex: String?           // tab bar 选中色（nil 则用 primary）
    let stickerTone: StickerTone      // 贴纸描边/底色微调，让布布贴纸融入主题
    let watermark: Watermark?         // 背景水印图案：.stars/.clouds/.petals，3% 透明度
}
```

落地要点：
- `BubuTheme.Color.card/softFill/hairline` 改为基于 `surfaceTintHex` 用 `Color.mix(_:with:by:)` 做 5–8% 色相偏移——**这一项做完，6 套主题立刻「套色彻底」**，是性价比最高的单点（DESIGN_UPGRADE P1 遗留，必做）；
- 全 App grep 一遍裸写的 `BubuTheme.Color.cream` / 固定 hex，凡属于「表面色」的统一走新 token；
- 语义色补全 `warning` / `info` 两枚（同步重试中 / AI 处理中），禁止页面裸写 `.orange/.blue`。

### 2.2 首页头部：MeshGradient 呼吸背景（高级感的核心一招）

项目 iOS 18+，可以直接用 SwiftUI 原生 `MeshGradient`：

- 首页 hero 区（头像 + 年龄区域）背景从「平直双色渐变」换成 3×3 `MeshGradient`，控制点取 `meshPalette`；
- 控制点做**极慢漂移**（单循环 12–20s，位移幅度 ≤ 0.08，`TimelineView(.animation(minimumInterval: 1/30))` 驱动即可，无需 120fps）——肉眼感觉是「光在纸上流动」，而非动画；
- `reduceMotion` 时静止；「布布照片」背景模式下不叠加 mesh，只保留暗角渐变保证文字对比度；
- 注意性能：mesh 只放 hero 一处，**不进滚动区**，实测 GPU 占用后再决定是否 `drawingGroup()`。

这一招 + 纸纹（下条）是「初期产品 → 高级产品」观感差异的大头。

### 2.3 纸感与材质

「时光机」应该有纸的温度，而不是手机屏的玻璃感：

- **纸纹**：制作 2 张极淡的可平铺噪点/纤维纹理（PNG，128×128，不引入第三方库），以 `overlay + .blendMode(.multiply) + opacity 0.03–0.05` 叠在卡片与背景上。深色主题（星夜）换用亮噪点 `.screen`；
- **信物 motif 体系化**：时间胶囊已有信封/蜡封隐喻，扩展成全 App 一致的「纸品语言」——
  - 时光轴卡片顶部可选「胶带」装饰（按 entry 心情随机 2° 内旋转，贴纸感）；
  - 里程碑点亮卡加「邮票齿孔」描边变体；
  - 导出的 HTML 档案同步套用纸纹背景（`ArchiveExporter` 模板里加同款 CSS）；
- **克制原则**：纹理透明度封顶 5%，姥姥模式（大字号）下自动关闭装饰元素，对比度永远优先。

### 2.4 新主题 + 季节系统

- 新增 2 套精品主题（宁缺毋滥）：
  - **「奶油绘本」**：米白纸底 + 棕墨主色 + serif 标题——给喜欢「素」的家长，最像实体日记本；
  - **「晚霞」**：珊瑚→紫的 mesh 渐变，偏成人审美，发朋友圈截图最好看；
- **季节自适应是模式不是主题**：设置里加「跟随季节」开关，按月份在现有主题里自动轮转（3–5 月薄荷绿、6–8 月晴空蓝、9–11 月珊瑚暖阳、12–2 月星夜），生日月强制「生日彩带」装饰层；
- **App 图标随主题**：`alternate app icons` 每套主题一枚 + 生日月专属（DESIGN_UPGRADE 远期项，并入本 Wave，成本就是出 8 张图）。

### 2.5 主题切换体验

现在 `ThemeSettingsView` 是色点列表，改成「试衣间」：

- 每套主题一张**迷你首页实景卡**（缩小版 hero + 统计卡 + 一条时光轴卡片，用真实组件渲染而非截图），横向分页滑动预览；
- 选中时全屏色彩以圆形揭幕扩散过渡（中心点 = 手指位置，0.5s，`mask + scale` 实现；reduceMotion 退化为 crossfade）；
- 切换成功 haptic `.success` + 布布贴纸 `.yeah` toast「换好啦」。

### 2.6 字与排版收尾

- 首页大年龄数字 bold + 单位 regular、`fontWidth(.expanded)`（DESIGN_UPGRADE 2.3 遗留）；
- 第一人称日记与胶囊信纸统一 serif（「布布的笔迹」一致性）；
- 全 App 数字场景 `monospacedDigit()` 查漏（成长报告 Charts 坐标轴、健康数值输入）。

**Wave J 专项验收**：6+2 套主题 × 浅/深色 × 「跟随季节」共 16 组截图无一处「旧暖棕残留」；hero mesh 在 iPhone 17 Pro 实机 GPU 帧时间 < 3ms；纹理开关后对比度复测 ≥ 4.5:1。

## 3. Wave I · 流畅度工程 —— 「动效帧率更好」

> 原则：**帧率是被管理的指标，不是被感觉的氛围**。本 Wave 先建测量，再修大头（图片管线），最后逐项过渲染预算。

### 3.1 先建基线（半天）

- 实机（ProMotion 120Hz）跑 Instruments **Animation Hitches** 模板，固定脚本：种子数据扩到 300 条 entry + 500 张图（`-uitest-seed` 加一档 `-uitest-seed-large`），录制「时光轴快速滚动 10 秒 / 进出详情 zoom 转场 ×5 / 胶囊开启三幕 / 主题切换」四个场景；
- 记录 hitch ratio（目标 **< 5 ms/s，理想 0**）存入本文附录表格，之后每个 Wave 结束复测一次——**没有基线的优化都是玄学**；
- DEBUG 加 `-uitest-fps` 启动参数：右上角浮动 FPS/hitch 小标（`CADisplayLink` 实现，几十行），日常开发肉眼盯。

### 3.2 图片管线重构（最大头，预计 2 天）

`MediaThumbnail.swift` 现状的四个问题逐一修：

| 问题 | 现状 | 改法 |
|------|------|------|
| 重复解码 | 每次 cell 复用重读磁盘 + `UIImage(data:)` | 新建 `ThumbnailProvider`（actor）：`NSCache<NSString, UIImage>` 内存缓存，key = `mediaId + 像素档位`，`totalCostLimit` 按图片字节数记账（约 150MB 上限，收到内存警告自动清） |
| 全尺寸解码 | `UIImage(data:)` 解出原始分辨率，绘制时主线程再缩 | 用 ImageIO `CGImageSourceCreateThumbnailAtIndex` + `kCGImageSourceThumbnailMaxPixelSize = 显示边长 × scale` 直接解到目标尺寸（`GrowthMoviePlayer` 已有同款代码，抽成共享工具）；解码后 `byPreparingForDisplay()` 预解码位图，杜绝首帧绘制卡顿 |
| 缺缩略图解原图 | thumbnail 缺失时直接解原图（十几 MB 的 HEIC） | 第一次遇到缺失就在后台生成缩略图**落盘补齐**（写回 `thumbnailFileName` 约定路径），下次直接命中；老数据启动后低优先级批量补 |
| 远端无缓存 | `AsyncImage` 每次重下 | 删掉 `AsyncImage` 分支：`SyncEngine` 本就会把远端媒体下载落地，过渡期改为「触发 MediaStore 下载 + 占位呼吸闪烁」，下载完走统一本地管线 |

附带两条：
- 照片墙网格（`HomeDestinations.swift`）与时光轴共享同一个 `ThumbnailProvider`，按网格 cell 尺寸请求小档位，**同图不同尺寸分开缓存**；
- 列表预取：`LazyVStack` 里对 `ForEach` 的下一屏 entry 调 `provider.prefetch(mediaIds:)`（`task(priority: .background)`），滚动到时已在缓存。

### 3.3 数据层与列表

- `TimelineView.groupedSections` 是计算属性 → 每次 body 求值 O(n) 重分组。改为 `@State` 缓存 + `.onChange(of: entries)` 时重算；300 条以上时分组移到后台再回主线程赋值；
- `@Query` 加 `fetchLimit` 起步 120 条 + 滚到底「再看更早的」追加（SwiftData 的 `FetchDescriptor.fetchOffset` 实现）；布布记到 5 岁时这页会有几千条，**现在就把分页机制铺好**；
- `ForEach(Array(section.entries.enumerated()))` 的 `enumerated` 包装破坏惰性且 index 仅用于错峰入场动画——首屏入场动画改为只对可见前 6 张做（已有约定），用 `entry.id` 直接当 identity，去掉 enumerated；
- 卡片内 `TimelineView(.periodic)`（胶囊倒计时）确认作用域只包住倒计时文本本身，而不是整张卡。

### 3.4 渲染预算（逐项过）

| 项 | 规则 |
|---|------|
| 阴影 | 滚动区内卡片阴影统一 `shadow(radius ≤ 8)` 且加 `.compositingGroup()`；禁止阴影叠阴影 |
| 玻璃/模糊 | 每屏 Liquid Glass 表面 ≤ 2 处（既有约定），滚动区内禁止实时 blur |
| 粒子 | 蜡封碎裂/星光保持 CAEmitter（GPU 合成），SwiftUI 端星光视图加 `drawingGroup()` |
| 波形 | `VoiceComponents` 录音波形若仍为 Timer 驱动，迁移 `TimelineView(.animation)` 或 `phaseAnimator`；`CADisplayLink` 场景设置 `preferredFrameRateRange(minimum: 30, maximum: 120, preferred: 60)`——波形不需要 120 |
| Mesh/纹理 | Wave J 的 mesh 只在 hero 静态区；纹理 overlay 是静态图平铺，零每帧成本 |
| 透明度动画 | 循环呼吸类（胶囊 24h 发光）确认只动 `shadow/opacity` 不触发布局 |

### 3.5 动效语言收尾（吸收 DESIGN_UPGRADE 余项）

- ✅ 已有 `BubuMotion` 五曲线 / `BubuHaptics` 六触觉，本 Wave 全局 grep 裸 `.spring(`/`.easeInOut(` 清零；
- 里程碑进度环入场 0→当前值动画（1s easeOut）+ 待点亮卡按压预览彩色（P1 遗留）；
- 时光轴卡片层级重排：媒体大图 > 正文 > 元信息三层，元信息整体降一级灰度（P1 遗留）；
- tab 图标统一 `.fill` 变体 + `symbolEffect(.bounce, value: selection)`（P2 项，5 分钟）；
- 滚动年份指示器（右侧拖动浮出「2025 · 1岁」气泡）—— 记录超过 200 条后才有价值，放 Wave L 与分页一起做。

**Wave I 专项验收**：300 条种子数据下，时光轴满速滚动 hitch ratio < 5ms/s（实机）；内存峰值 < 400MB；任何图片首次出现到清晰 ≤ 200ms（缓存命中 ≤ 16ms）；快速滚动时无空白闪烁。

## 4. Wave K · 互动系统 —— 「互动更强」

> 现状的互动是「各自记录 + 合奏补充」，是**协作**不是**互动**。这一 Wave 让家人之间有来有回，并把触点伸到 App 外（Widget/通知）。隐私底线不变：一切互动数据走自家 PocketBase，无任何第三方。

### 4.1 亲一下：轻量 Reaction（P0，互动的最小闭环）

- 时光轴卡片/详情页**长按 → 弹出 4 枚反应**：❤️ 亲亲 / 🤗 抱抱 / 😂 笑死 / 😭 感动（克制，不做全 emoji 面板）；
- 数据上复用 `Comment` 模型加 `kind: .reaction` 枚举位（迁移成本最低，同步链路现成），同一人同一条只保留最新反应；
- 动效：长按弹出用 `quick` 曲线，选中后心形从手指位置上浮飘散（3–5 个粒子，0.8s）+ `.selection` 触觉；
- 卡片元信息行尾显示聚合「❤️×2 🤗×1」；对方设备同步到新反应时，`FeedEvent` 记一条「姥姥亲了『布布第一次扶站』」；
- **为什么值得做**：姥姥可能不会打字评论，但一定会点亲亲——这是适老化的互动形式。

### 4.2 评论升级：语音回复

- `CommentComposeSheet` 加语音条目（复用 `AudioRecorder`/`VoiceComponents`，全部现成）——「对这一刻说句话」；
- 语音评论自动归入「成长之声」的「家人对她说的话」档案（`VoiceMemo` 链路已有，打通归档即可）；
- 详情页评论区语音条带波形 + 时长，点击播放。

### 4.3 每日一问（让「不知道记什么」消失）

- 内置 100 条本地问题库（按布布月龄分桶）：「今天布布最像哪种小动物？」「她今天发明了什么新词？」「她现在最讨厌什么？」——以日期为种子轮换，全家同一天看到同一题；
- 首页「记录此刻」卡下方加一张**今日问题卡**（贴纸 `.thinking` + 问题 + 「答一句」按钮），点击进快速记录页并预填问题为引导语，保存的 entry 自动打「问答」标签；
- 多人都答了同一题时，时光轴把当天的答案**并排成对话卡**——爸爸说/妈妈说/姥姥说，这是「家人合奏」的日常化入口；
- 零后端、零 AI 依赖，纯本地题库 + 现有同步。

### 4.4 家庭周报（把回顾变成仪式）

- 每周日晚 8 点本地通知：「布布的这一周 →」，点开进**周报页**：本周 N 个瞬间 / 最佳照片（按反应数选）/ 谁记得最多（排行榜，第一名头像戴小皇冠）/ 解锁的里程碑 / 下周的「第一次」预告（成长洞察现成能力）；
- 周报页底部「生成分享卡」：渲染成一张竖版长图（`ImageRenderer`），发家庭群——**这是产品自然生长出圈的口子**；
- AI 可用时附一段 50 字周记（`BubuAIService` 加一个 prompt，降级为模板文案）。

### 4.5 Widget 全家桶（App 外触点，P0）

- WidgetKit 扩展，三档：
  - **小**：布布贴纸 + 「来到世界第 N 天」实时年龄；
  - **中**：「那年今日」照片 + 年龄角标（无照片时显示今日问题）；
  - **锁屏行内**：「距 2 岁生日还有 151 天」；
- 中号 Widget 加 `AppIntent` 交互按钮「记一笔」深链到快速记录页；
- **工程注意**：SwiftData store 需迁入 App Group 容器（`ModelConfiguration(groupContainer:)`），老数据做一次性文件搬迁——这是本 Wave 唯一有风险的改动，先写迁移测试再动手；Widget 只读，避免双进程写冲突。

### 4.6 仪式时刻的「共时性」

- 时间胶囊到期：每台设备本地 `ReminderScheduler` 在 `unlockAt` 各自排通知（胶囊本身已同步），文案「有一封等了 6 年的信，今天可以拆了」——**全家同一时刻收到**，开启典礼变成家庭事件；
- 里程碑点亮后，其他成员设备下次同步到时弹轻提示 +「去亲一下」快捷按钮（串联 4.1）；
- 生日当天：打开 App 触发一次性彩带雨（CAEmitter，4s，当年仅一次，reduceMotion 静态彩带横幅）。

**Wave K 专项验收**：三台真机互发反应/语音评论 10 分钟内收敛一致；Widget 在断网时显示正确（读 App Group 快照而非等同步）；App Group 迁移在升级安装路径上零数据丢失（写专项单测 + 真机覆盖安装验证）。

## 5. Wave L · 功能完善与合理化 —— 「更完善、更合理」

### 5.1 设置页重构（用户点名，P0）

**问题本质不是皮肤，是信息架构**：高频（身份/主题/提醒）与一年一次（服务器/AI Key）混排。重构为「**个人化在前，资料中间，机房收进柜子**」：

```
设置（自绘卡片组，弃系统 Form 平铺）
│
├─ 顶卡：当前身份（大头像 + 名字 + 「点我换人」）       ← 适老第一入口
│        右上角同步状态徽章（绿点「已同步」/转圈/灰「离线」）
│
├─ 「布布」卡组：档案 · 成长之声 · 健康记录
├─ 「这个家」卡组：家庭成员 · 每周报告开关
├─ 「外观」卡组：主题与外观 · 跟随季节 · App 图标
├─ 「提醒」卡组：那年今日 · 每日一问 · 周报时间
├─ 「数据」卡组：
│    ├─ 备份健康度卡（见 5.5：上次同步/媒体数/占用/上次导出）★ 新
│    └─ 导出全量档案
│
├─ 「高级 · 自托管」→ 二级页（默认折叠收纳）：
│    服务器配置 / AI 服务配置 / 连接诊断（把 LabeledContent 状态、
│    测试按钮、失败原因全部收到这里，普通成员永远不用看见 URL 和密钥）
│
└─ 页脚：版本号 + 「为布布而做，2025 ❤️」
```

设计细节：
- 卡组用 `bubuGlassSurface`/实色卡 + 行高 ≥ 52pt + SF Symbol 统一上色圆底（iOS 系统设置那种彩色小方块的暖色版）；
- 同步状态从 9 行 LabeledContent 压缩成**一颗徽章 + 点开的诊断页**——状态正常时不该占任何注意力；
- 「连接测试并保存」这类工程师文案改口语：「试试能不能连上」「通啦 ✓」（部分已是，统一走查）；
- 高级页入口加锁图标 + 副标题「给装服务器的那个人」。

### 5.2 全局搜索（记录超过百条后的刚需）

- 时光轴导航栏加 `.searchable`：命中范围 = 正文/标签/地点/作者/心情/「第一次」名称；
- 搜索词为空时展示**浏览维度**：按标签云 / 按地点 / 按心情 / 按月龄段（「1 岁 0–6 个月」）的快捷胶囊；
- 实现：SwiftData `#Predicate` 组合 + 结果按月分组复用时光轴卡片；标签云数据启动时后台聚合缓存；
- 远期（Wave M）：接 Vision 已产出的图片标签做「搜图」——数据早就在 `Media` 上，只差进搜索谓词。

### 5.3 「那年今日」从通知变成页面

- 通知有了、首页回忆卡有了（P0 已做），缺**落地页**：按年份纵向排「2025 的今天 / 2024 的今天」，每年一张大卡（照片 + 当时年龄 + 当时记录原文）；
- 入口：首页回忆卡点进 + 通知深链；
- 空年份显示「这一年的今天没有记录——今天补一条？」（贴纸 `.bye` + 直达记录按钮）——把怀旧转化为新记录。

### 5.4 健康页：从流水账到成长曲线

`HealthRecord` 模型已有，缺呈现：
- **WHO 生长标准曲线**（0–5 岁身高/体重/头围百分位，标准数据内置 JSON，约 30KB，无网络依赖）：Swift Charts 画 P3/P15/P50/P85/P97 五条带 + 布布的实测点连线，落点处标注「P62 · 比同龄 62% 的小朋友高」；
- 录入优化：身高体重输入用大号数字滚轮 + 上次值预填；支持补录历史日期；
- 疫苗接种表：内置国家免疫规划程序表（一类苗），按月龄自动排期 + 完成打卡 + 提前 3 天提醒（`ReminderScheduler` 扩一个类别）；
- 就医/用药记录已有字段的，详情页时间线化展示。

### 5.5 数据安全兜底（这个产品最不能输的地方）

- **备份健康度卡**（设置「数据」卡组）：四项体检——上次成功同步时间 / 待同步项 / 上次全量导出距今天数 / 服务器磁盘可用空间（PocketBase 加一个 `/health` 自定义端点返回磁盘信息）。任一项超阈值（如 90 天没导出）首页同步胶囊变 `warning` 色提醒；
- **真 E2E 时间胶囊 v3**（HANDOFF 未做清单第 1 位）：随机对称密钥加密内容，密钥存 iCloud Keychain（`kSecAttrSynchronizable`）+ 生成 24 词恢复码引导打印/抄写收进实体盒子；旧 v2 信件保持兼容读取；文案同步升级为「真加密」；
- **导出提醒自动化**：每季度第一个周末通知「该给布布的档案存个底了」，导出完成后记录时间戳进健康度卡。

### 5.6 PDF 年册导出

- `ArchiveExporter` 旁新增 `YearbookExporter`：选一个年龄段（如「1 岁这一年」），生成 A4 竖版 PDF——封面（姓名/年龄/精选照片）+ 月份章节页 + 照片网格 + 摘录的记录原文 + 里程碑页 + 家人寄语页（取语音评论的转写）；
- 用 `ImageRenderer` 逐页渲染 SwiftUI 视图进 `UIGraphicsPDFRenderer`，零第三方依赖；
- 排版套用当前主题色 + 纸纹——**这是「30 年后还能摸到」的实体交付物**，也是送爷爷奶奶的年度礼物。

### 5.7 记录体验补完

- 视频录制：快速记录页现在只有拍照/录音/文字 + 相册补充，加直接录像（`CameraCaptureView` 扩展，限 60s，姥姥手抖防误触：录制中大停止按钮）；
- 批量补录：相册多选导入时按 EXIF 日期**自动拆成多条** entry（现在是合并一条），逐条确认的批量流程；
- 草稿保护：快速记录页意外退出（来电/切后台）自动存草稿，下次进入恢复。

**Wave L 专项验收**：设置页可用性测试——「给姥姥的三道题」：换身份 ≤ 2 步、改主题 ≤ 3 步、永远找不到也不需要找到服务器配置；成长曲线与 WHO 官方表抽查 5 点误差 < 1%；PDF 年册 200 条记录生成 < 30s、文件 < 80MB。

### 6.3 贴纸资产扩容

## 9. 工程保障（升级期间新增的护栏）

1. **性能门禁**：附录基线表进仓库，每 Wave 结束实机复测填表；hitch ratio 回退超 20% 不准合入下一 Wave；
2. **截图回归**：`-uitest-*` 启动参数体系已成熟，补一个 `screenshots.sh`：6+2 主题 × 浅深 × 5 个核心页自动出图到 `screenshots_regression/`，肉眼 diff（不引 snapshot 测试库，保持零依赖）；
3. **迁移类改动先写测试**：App Group 搬迁、胶囊 v3、缩略图补齐三处涉及存量数据，照 `CapsuleCryptoTests` 的先例先写回归再动手；
4. **每个独立功能一个中文 commit**（仓库惯例），Wave 完成打 tag `wave-i` … `wave-l`；
5. **适老验收常态化**：每 Wave 结束跑一遍「姥姥三道题」+ 最大字号 + reduceMotion + VoiceOver 抽查首页和记录页。

## 10. 明确不做（延续并新增）
